Bogi Tracker — Guide

Trace Mapei (couleur = vitesse)

Chaque segment entre deux points YB est coloré selon la vitesse moyenne : bleu ≈ 0 kn → cyan → vert ≈ 17 kn → jaune → rouge ≥ 35 kn. Les segments gris translucides attendent encore leur détail (enrichissement en arrière-plan, quelques minutes après chaque boot serveur).

Points YB cliquables

Chaque point blanc sur la trace = une position envoyée par Yellow Brick (~1 toutes les 15 min). Clic = popup avec date/heure, vitesse, cap, température eau et coordonnées exactes. Toggleable depuis le menu.

AIS (cibles + traces)

Bateaux à 100 NM autour de Mapei, reçus via aisstream.io. Triangles bleu-vert pour chaque cible. Une trace de la dernière heure est dessinée si plusieurs positions sont reçues.

Le nombre de cibles varie naturellement : Mapei avance vite (20+ kn), le bbox 100 NM se déplace avec elle, des bateaux sortent du rayon et d'autres entrent. La fenêtre est glissante (1h) : si un bateau n'émet plus pendant 1h il disparaît du tracker (mais il existe toujours, juste pas dans nos données).

Extrapolation +1h / +5h / +10h (rouge)

Trois badges rouges à la position projetée en supposant cap et vitesse constants depuis le dernier point YB. Pointillé rouge reliant la position actuelle au +10h. Toggleable.

Curseur timeline Windy (orange)

Quand tu fais glisser la barre de temps de Windy en bas de l'écran, le badge orange « @HH:MM » suit en temps réel : il indique la position où Mapei serait à cette heure-là (même hypothèse cap+vitesse constants), pour visualiser le bateau dans la météo projetée. Toggleable avec l'extrapolation.

Vent et fond de carte

Géré par Windy : bouton « Vent » en haut à droite pour changer l'overlay (vent / temp / pression / vagues / etc), menu burger Windy pour basemap satellite / standard / dark / etc. Tes préférences sont mémorisées par Windy.

Auth et préférences

Mot de passe partagé équipage, session conservée 30 jours via cookie. Tes toggles UI (AIS, points, extrapolation) sont mémorisés par navigateur. Pas de compte individuel.
